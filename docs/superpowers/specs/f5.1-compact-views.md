# Compact Views — Design Spec

**Date**: 2026-05-26
**Status**: Approved (brainstorming phase)
**Scope**: Schedule, Groups, Knockout (Bracket) — frontend only

## 1. Problem

Today the three main fixture views (`ScheduleView`, `GroupView`, `BracketView`) only render in "expanded" mode: large match cards stacked vertically, group tabs and round tabs, a full standings table on top of the group view. On mobile this means a lot of vertical scroll, and the knockout view never shows the tournament shape — only one round at a time.

Goal: introduce a global "compact" view mode that:

- shows each match as a single horizontal row in Schedule and Groups,
- moves the standings table from above to a collapsible block (so matches breathe),
- replaces the round tabs in the bracket with a full **bracket tree** (à la classic World Cup print bracket) where past rounds can be collapsed to reduce noise.

Expanded mode stays as-is for users who prefer it.

## 2. Non-goals

- No changes to data model, reducer actions, Nostr sync, scoring, or export/import.
- No new UI library. Stays vanilla CSS.
- No changes to Ranking, Simulator, Settings, How-To-Play. Only the three fixture views.
- No migration of existing localStorage keys (the new `viewMode` key is fresh).

## 3. Decisions (from brainstorming)

| Aspect | Decision |
|---|---|
| Toggle scope | Global, persistent across all three views |
| Toggle UI | Icon button in TopBar |
| Default on first load | `expanded` (zero disruption for current users) |
| Schedule compact layout | `[stage+time badge] [team right-aligned] [score in] – [score in] [team left-aligned]`, day separator inline horizontal with hairlines |
| Groups compact layout | Collapsible standings table on top (4 cols: team / PJ / DG / Pts), then compact match rows below — same row component as Schedule |
| Bracket compact layout | Bracket tree (5 columns: R32 / R16 / QF / SF / F) replacing the round tabs. 3P shown as an extra card below the Final in the same column |
| Bracket past-round collapse | Auto-collapse when **all** matches in a round have a score; user can override manually; override persists in `sessionStorage` |
| Bracket score editing | Inline mini inputs — reuses existing `ScoreInput` with a CSS-only `compact` variant. Penalty picker and prediction indicator continue working |

## 4. Architecture

### 4.1 Preference storage

A new local-only preference, separate from `FixtureState`. Not synced via Nostr, not part of export/import.

```ts
// src/utils/viewMode.ts
export type ViewMode = "expanded" | "compact";
const KEY = "viewMode";
export function loadViewMode(): ViewMode { /* "expanded" if missing or invalid */ }
export function saveViewMode(mode: ViewMode): void;
```

### 4.2 Provider

```tsx
// src/context/ViewModeContext.tsx
useViewMode(): { mode: ViewMode; toggle: () => void; setMode: (m: ViewMode) => void };
```

No reducer (single value, no derived state). Reads on mount, writes on each change.

### 4.3 Provider stack (main.tsx)

```
LocaleProvider
  ViewModeProvider          ← NEW
    NostrProvider
      FixtureProvider
        App
```

`ViewModeProvider` sits below `LocaleProvider` because it doesn't depend on locale; it sits above `NostrProvider` because nothing in the Nostr or Fixture layer reads it (only the UI components do).

### 4.4 New / modified files

```
src/
├── utils/viewMode.ts                  # NEW
├── context/ViewModeContext.tsx        # NEW
├── components/
│   ├── TopBar.tsx                     # MOD — toggle button
│   ├── TopBar.css                     # MOD
│   ├── ScheduleView.tsx               # MOD — branch on mode
│   ├── ScheduleView.css               # MOD — .schedule-compact-* classes
│   ├── GroupView.tsx                  # MOD — branch on mode + collapsible table
│   ├── GroupView.css                  # MOD
│   ├── BracketView.tsx                # MOD — render BracketTree in compact mode
│   ├── BracketView.css                # MOD

│   └── CompactMatchRow.css            # NEW

```

### 4.5 CompactMatchRow shape

```
[badge: stage + time (small)]  [team home right-aligned]  [in] – [in]  [team away left-aligned]
                                                          locked / synced / indicator badges
```

Props mirror what `MatchCard`/`ScheduleMatchCard` already receive. Lives in one component so behavior stays consistent across Schedule and Groups.

The bracket compact card is **not** a `CompactMatchRow` — it's a smaller vertical card (~110px wide) so it gets its own sub-component inside `BracketTree.tsx`.

## 5. Data flow

```
User clicks TopBar toggle
  → useViewMode().toggle()
  → ViewModeContext setState(next)
  → saveViewMode(next)
  → Re-render: ScheduleView / GroupView / BracketView read mode and branch
```

No reducer action. No dispatch. No effect on `FixtureState`.

## 6. View-by-view behavior

### 6.1 ScheduleView compact

- Same `matchesByDay` grouping (unchanged logic).
- Day header is replaced by an inline horizontal separator: thin hairline + uppercase date label + thin hairline.
- Each match renders as `<CompactMatchRow />`.
- All existing behavior preserved: lock state, sync badges, auto-sync tooltip, prediction indicator, `SET_GROUP_SCORE` / `SET_KNOCKOUT_SCORE` dispatch.

### 6.2 GroupView compact

- Group tabs (A–L) unchanged.
- Title `groups.title` unchanged.
- Standings: new `<CompactStandings />` component:
  - 4 columns: team (flag + abbrev), PJ, DG, Pts.
  - Collapsible via local `useState`, **expanded by default** on desktop, **collapsed by default** on mobile (`window.matchMedia("(max-width: 768px)").matches` on mount).
  - Chevron clickable in the header bar.
- Match list below: array of `<CompactMatchRow />`, same layout/behavior as Schedule.

### 6.3 BracketView compact

- Round tabs (R32 / R16 / QF / SF / F) **hidden**.
- `<BracketTree />` renders 5 columns side by side.
- Column header: round label (translated via `knockout.rounds.${r}`) + completion check (`✓`) when round is fully scored + chevron for manual toggle.
- Expanded column: vertical stack of mini-cards, one per match in that round, in `matches.sort((a, b) => a.dateUtc.localeCompare(b.dateUtc))` order.
- Collapsed column: width clamps to 32px, label rotates with `writing-mode: vertical-rl`, matches hidden.
- F column: extra small card labeled "3P" stacked below the Final card.
- Each mini-card embeds `<ScoreInput />` (the existing component) wrapped in a `.bracket-tree` class for the compact CSS variant. No changes to `ScoreInput.tsx` itself.

### 6.4 Auto-collapse logic

```ts
// pure, testable
function computeAutoCollapsed(
  matchesByRound: Record<KnockoutRound, ResolvedKnockoutMatch[]>,
  mode: "predictions" | "actual"
): Set<KnockoutRound> {
  const field = mode === "predictions" ? "prediction" : "result";
  const collapsed = new Set<KnockoutRound>();
  for (const [round, ms] of Object.entries(matchesByRound)) {
    if (ms.length > 0 && ms.every((m) => m[field] !== null)) {
      collapsed.add(round as KnockoutRound);
    }
  }
  return collapsed;
}
```

The F column is "complete" only when both F and 3P have scores (both are in `matchesByRound.F` ∪ `matchesByRound["3P"]` for collapse purposes; spec treats them as one column).

### 6.5 Manual override persistence

`sessionStorage.bracketCollapseOverrides` shape:

```ts
type Override = Partial<Record<KnockoutRound, "expanded" | "collapsed">>;
```

If a key exists, its value wins over auto-collapse. Cleared at session end (browser tab close) — intentional, lighter than `localStorage` for an ephemeral UI preference.

## 7. Styling

### 7.1 Layout

Bracket tree:
```css
.bracket-tree {
  display: grid;
  grid-template-columns: repeat(5, minmax(110px, 1fr));
  gap: 12px;
  overflow-x: auto;
  padding: 4px 0;
}
.bracket-tree .col.collapsed { width: 32px; padding: 4px; }
.bracket-tree .col.collapsed .col-title { writing-mode: vertical-rl; transform: rotate(180deg); }
```

Compact ScoreInput:
```css
.bracket-tree .score-field { width: 28px; height: 24px; font-size: 12px; }
.bracket-tree .penalties-pick { width: 24px; height: 24px; font-size: 14px; }
```

### 7.2 Mobile (<768px)

- Bracket tree: horizontal scroll. Minimum viable width ≈ 4 cards × 110px + 1 collapsed × 32px ≈ 472px including gaps; comfortable horizontal scroll on a 360px viewport.
- Tap targets in compact rows: minimum 32px height.
- TopBar toggle: visible at all viewport widths.

### 7.3 Animations

| Element | Animation |
|---|---|
| Mode toggle | None (instant render switch) |
| Bracket column collapse | `transition: width 200ms ease, opacity 200ms` |
| Standings table collapse | `transition: max-height 200ms ease, opacity 200ms` |
| `prefers-reduced-motion: reduce` | All transitions disabled |

## 9. Edge cases

1. **Bracket in predictions mode with no predictions**: every score null → no round auto-collapses → all expanded. Correct.
2. **Bracket in actual mode, tournament hasn't started**: same as above.
3. **R32 complete, R16 in progress**: R32 auto-collapses, R16 stays expanded.
4. **F complete but 3P pending (or vice versa)**: F column treated as complete only when both F and 3P have scores.
5. **Lock time gate (1h before kickoff)**: `ScoreInput` already receives `locked` and `disabled` props — no changes in compact, behavior preserved.
6. **Auto-sync results (actual mode)**: `CompactMatchRow` must receive `editable`, `syncedResultIds`, and `autoSyncedAt` tooltip in the same way the current cards do.
7. **Rapid mode toggle during score edit**: `ScoreInput`'s `useEffect` resyncs its local input string with the `score` prop on prop change. Already-committed values survive; uncommitted in-flight typing is discarded (acceptable trade-off).
8. **`prefers-reduced-motion: reduce`**: collapse transitions disabled, instant change.

## 10. Testing strategy

Per `CLAUDE.md`: "Components are not [tested] — integration is verified via `pnpm run build` + manual smoke in dev." Tests apply to pure logic only.

New unit tests:

`src/utils/__tests__/viewMode.test.ts`:
- `loadViewMode()` returns `"expanded"` when localStorage is empty.
- `loadViewMode()` returns `"expanded"` when localStorage value is invalid (not `"expanded"` or `"compact"`).
- `loadViewMode()` returns the stored value when valid.
- `saveViewMode(mode)` writes to localStorage under `viewMode`.

`src/components/__tests__/bracketAutoCollapse.test.ts` (the pure function extracted from `BracketTree.tsx`):
- Returns empty set when all matches have null scores.
- Returns the rounds whose matches all have scores (`predictions` field check).
- In `actual` mode uses `result` field, not `prediction`.
- F + 3P are merged: F is "complete" only when both have scores.

i18n consistency tests already in the repo will catch any missing key in EN/PT vs ES.

Manual smoke checklist (in `pnpm run dev`):
- Toggle from TopBar; persistence survives reload.
- Schedule compact: edit a score, indicator updates.
- Groups compact: collapse/expand the standings table; switch group tab.
- Bracket compact: collapse a round manually; refresh page → override survives within session; close tab and reopen → override gone (auto-collapse takes over).
- Bracket compact: edit a knockout score; tie a knockout; verify penalty picker is reachable and works inline.
- Mobile viewport (375px in DevTools): all three views functional, horizontal scroll in bracket only.

## 11. Verification gates

- `pnpm test` — all existing + new tests pass.
- `pnpm lint` — does not add new errors (the existing 9 pre-existing errors are tolerated, as documented in CLAUDE.md).
- `pnpm run build` — completes without TS errors.

## 12. Out of scope (explicit, to avoid scope creep)

- Per-view toggles (decided: global only).
- Auto-detection of viewport for default mode (decided: always expanded as default).
- Compact mode for Ranking / Simulator / Settings / Help (decided: only the three fixture views).
- Animations between modes (decided: instant switch).
- Migration to a UI library or design system.
