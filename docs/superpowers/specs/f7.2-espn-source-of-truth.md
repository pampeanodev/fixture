# ESPN as the Source of Truth

## 1. Problem

Two days into the tournament, the f7.1 conflict model ("API only fills voids; admin wins by existing first") failed in production:

- A room admin's device accumulated wrong manual results (entered while the spurious circuit-breaker trip had unlocked editing). The admin push overwrote every member's correct ESPN data, and auto-sync couldn't repair it — fill-voids never touches an existing result.
- The recovery path ("admin clears the score, auto-sync refills") requires the *polluted* device to act correctly. In practice the poison propagated faster than the cure.

The owner's call (2026-06-12): official results must never be hand-typed while the API works, and ESPN must win every conflict — including against the admin.

## 2. Goals

- A wrong result anywhere in a room converges to ESPN's value within one auto-sync tick (30 min), with no human action.
- Manual result entry is impossible while the API is healthy.

## 3. Non-goals

- Out of scope: an admin escape hatch to override ESPN. Decision: "ESPN es la verdad, punto". If ESPN is wrong, the fixture is wrong until ESPN corrects (each tick re-applies, so the correction propagates alone).
- Out of scope: removing the admin push subsystem entirely (it survives as a fallback).

## 4. Design

Priority order per match result, highest first:

1. **ESPN auto-sync** — always on (settings toggle removed), `APPLY_AUTOSYNC_RESULTS` overwrites whatever exists. Re-applies on every tick when values differ; idempotent no-op otherwise. Guards: terminal status, kickoff passed (`resultsGuard`), not during simulation.
2. **Admin push** — `APPLY_SYNCED_RESULTS` becomes **fill-voids only**. It covers members whose device can't reach ESPN but can reach Nostr relays, and distributes grace-period manual entries during an API outage. Still strips premature results; still clears its own entries when they vanish from the payload; never touches a result it didn't create.
3. **Manual entry** — gated by `graceLock` with the `autoSyncEnabled` escape removed: editable only when the circuit breaker is tripped or the grace period (3 h group / 4.5 h KO) elapsed without an API result. When the API recovers, tier 1 overwrites and the room converges.

## 5. Trade-offs

- *Toggle as escape hatch* (rejected): keeps a human override but also keeps the failure mode — any device with manual access can poison a room faster than ESPN heals it.
- *Admin-flagged overrides* (rejected): per-match "manual override" state adds UI + protocol surface to protect against ESPN errors, which are rarer and shorter-lived than human ones.
- Accepted cost: a wrong ESPN final sticks until ESPN fixes it. Penalty-shootout data quality now rests entirely on ESPN.

## 7. Implementation notes

Shipped together with this spec (same commit series):

- Reducer: `APPLY_AUTOSYNC_RESULTS` overwrites; `APPLY_SYNCED_RESULTS` fills voids and only tags matches it filled.
- `useAutoResultSync`: diff-based re-apply, `simulationActive` gate, stable 30-min interval (the tick no longer re-fires on every state change).
- `graceLock`: `EditabilityContext.autoSyncEnabled` removed; `loadAutoSyncEnabled`/`saveAutoSyncEnabled` deleted.
- Related hardening in the same incident: `randomizePredictions` respects locked matches and asks for confirmation; published commitment hashes are persisted per room and reused for locked matches (`f2` commit-reveal: a post-lock prediction mutation can no longer rewrite the published hash).
